Part 1: Filters and Edges

1.1. Convolution Implementation (Numpy vs. Scipy)

We implemented the 2D convolution operation using pure NumPy and compared its performance and behavior against the optimized scipy.signal.convolve2d.

Comparison Summary

  • Runtime: The NumPy-only implementation demonstrated $O(H \cdot W \cdot K^2)$ complexity, showing significantly slower runtime compared to Scipy's optimized Fast Fourier Transform (FFT) based convolution, especially for large images or large kernels.
  • Boundary Handling: My NumPy implementation uses zero-padding (explicitly adding zero rows/columns), while Scipy's default mode ('full') handles boundaries by implicitly extending the input with the specified boundary condition. This difference in padding must be accounted for to ensure output size consistency.

1.2. Partial Derivatives, Gradient, and Edges

Using finite difference filters (e.g., $ [-1, 0, 1] $), we computed the image gradient, magnitude, and binarized edges.

Edge Threshold Justification

We chose a threshold of $0.1$ for binarization. This value represents a minor tradeoff: it effectively removes low-magnitude noise edges (reducing false positives) while still retaining faint but important structural details (avoiding false negatives) in the cameraman's jacket texture.

1.3. Gaussian and Difference of Gaussian (DoG) Filters

We constructed Gaussian filters using cv2.getGaussianKernel and built DoG filters by subtracting two Gaussians with different $\sigma$ values.

Filter Visualizations and Applications ($\sigma=3$ for Gaussian, $\sigma_1=1, \sigma_2=5$ for DoG)

Comparison of Edge Detection

The Finite Difference method is highly sensitive to noise, producing thick, noisy edges. The DoG filter, due to its band-pass nature, inherently incorporates smoothing, resulting in thinner, cleaner, and better localized edges. This supports the notion that Gaussian pre-smoothing is crucial for robust edge detection.

Bells and Whistles: Gradient Orientations

We computed the gradient orientation $\theta$ for every pixel using the arctangent function $ \theta = \arctan2(\frac{\partial I}{\partial y}, \frac{\partial I}{\partial x}) $ without relying on built-in angle functions. Instead, we manually implemented a mapping to convert the ratio of partial derivatives into an angle in the range $[0, 2\pi)$.

Process Explanation

  • The $I_x$ and $I_y$ derivatives define the vector direction.
  • The $\arctan2(y, x)$ function (which was implemented manually using conditional checks on the signs of $I_x$ and $I_y$ to determine the correct quadrant) gives the angle.
  • Visualization: The resulting angles were mapped to a Hue-Saturation-Value (HSV) color space where Hue represents the angle (e.g., $0$ to $2\pi$ maps to $0$ to $360^{\circ}$) and Value represents the gradient magnitude.

Part 2: Applications

2.1. Unsharp Masking

The Unsharp Mask filter works by generating a high-frequency detail layer (the 'mask') and adding it back to the original image. The process is:

Sharpened = Original + $\alpha$ * (Original - Blurred)

The blurred version is obtained via a Gaussian filter, which removes high frequencies. Subtracting the blurred image from the original isolates the high-frequency components. Multiplying this by the amplification factor $\alpha$ controls the sharpening intensity.

2.3 + 2.4. Multi-resolution Blending

Multi-resolution blending uses Laplacian stacks to decompose two input images, $I_A$ and $I_B$, and a Gaussian stack of a mask $M$ to blend the images at each frequency level, resulting in a seamless composite image.

Blended Stack Level $l = L_{A,l} \cdot G_{M,l} + L_{B,l} \cdot (1 - G_{M,l})$

Bells and Whistles: Blending Exploration & Justification

For the irregular mask blend (e.g., a hand-drawn or generated contour), the boundary definition was critical. We found that the blending result is significantly improved when the mask contour is soft and aligned with the high-contrast edges of the images being blended, as the Gaussian stack does the heavy lifting of feathering the transition. We experimented with different $\sigma$ values for the mask's Gaussian stack, finding a large $\sigma$ ($>10$) provides the best, most imperceptible blend across the transition zone.
